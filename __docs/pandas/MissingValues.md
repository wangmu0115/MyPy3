## 1. 处理缺失值的方法

识别数据表中缺失值的方法一般可以分为两种：

- 通过一个覆盖全局的**掩码 mask** 表示缺失值；
- 通过一个**哨兵值 sentinel value** 表示缺失值。

使用单独的掩码数组会额外增加一个布尔类型的数组，从而增加存储与计算负担；哨兵值方法缩小了可以被表示为有效值的范围，可能需要在 CPU 或 GPU 算术逻辑单元中增加额外的计算逻辑。大多数情况下是不存在最佳选择的，不同的编程语言与系统使用不同的方法。

## 2. pandas 的缺失值

pandas 中有两种存储和处理空值的模式：

- 默认模式使用哨兵值的缺失数据方案，哨兵值为 `NaN` 或 `None`，取决于数据类型。
- 使用 pandas 提供的可空值数据类型，会创建一个附带的掩码数据来跟踪缺失数据，这些缺失数据会以特殊的 `pd.NA` 值呈现。

### 2.1. `None` 作为哨兵值

`None`是Python对象，任何包含None的数组类型都会被设置为`dtype=object`，表示这个数组是由Python对象构成的，在常见的快捷操作时，会消耗更多的资源。由于Python不支持None的算术运算，使用类似sum, max 这样的聚合运算时会导致出错。

pandas在数值数组中不使用None作为哨兵值。

NaN，数值类型的缺失值

`math.nan` 或 `np.nan`

无论与NaN进行任何操作最终结果都是 NaN
NaN的缺点是，它只是一个浮点数值，对于整数、字符串或其他数据类型，没有等效的 NaN 值。

为数组设置nan值的话，会自动为数组选择一个原生浮点类型。

虽然 NaN 和 None 有不同的用处，不过 pandas 将它们看成是可以等价交换的，在适当的时候会将两者进行替换。下表展示 pandas 对不同类型缺失值的转换规则

| 类型 | 缺失值转换规则 | NA 哨兵值|
|-----|--------------|---------|
|浮点型| 无变化|np.nan|
|对象类型|无变化|None 或 np.nan|
|整数类型|强制转换为float64|np.nan|
|布尔类型|强制转换为object|None 或 np.nan|
